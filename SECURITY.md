# Security Policy

## Supported versions

Only the latest release on the `main` branch is actively supported with security updates.

## Reporting a vulnerability

If you discover a security vulnerability in OpenSessionLog, please report it privately rather than opening a public issue.

You can:

- Open a [GitHub security advisory](https://github.com/opensessionlog/opensessionlog/security/advisories/new)
- Email the maintainers directly

We will respond as quickly as possible and coordinate a fix and disclosure.

## What to include

- A description of the vulnerability
- Steps to reproduce
- The version or commit where you observed it
- Any suggested fix or mitigation
